Project 1: The "Zero Trust" Wireless Onboarding System

Objective

To design and build a fully automated, highly secure wireless onboarding system for a new corporate environment. The system will move beyond simple guest access and implement "Zero Trust" principles, where no device is trusted by default. The goal is to create a seamless, secure experience for both BYOD and corporate-owned devices.

Core Technologies

  • hostapd & FreeRADIUS
  • Captive Portal Technology (Nodogsplash)
  • Certificate Authority (OpenSSL)
  • Automation Scripting (Python/Bash)

Project Manager Responsibilities

The graduate student PM will architect the entire onboarding workflow, design the RADIUS policy logic for dynamic VLAN assignment, develop automation scripts, and oversee all integration and testing.

Undergraduate Team Roles

  • PKI Specialist: Manages the Certificate Authority and certificate lifecycle.
  • Portal Developer: Customizes the captive portal and user-facing experience.
  • Network Tester: Performs validation testing to ensure correct policy enforcement.

Project 2: The WIDS/WIPS Comparative Analysis Framework

Objective

To create and document a standardized framework for testing and evaluating the effectiveness of different Wireless Intrusion Detection/Prevention Systems (WIDS/WIPS). The project involves comparing a commercial solution (like Extreme Networks AirDefense) against a custom-built open-source solution.

Core Technologies

  • ExtremeCloud IQ / AirDefense
  • Kali Linux Attack Suite (Aircrack-ng, bettercap)
  • Open-Source IDS (Kismet, Snort)

Project Manager Responsibilities

The PM will design the "Test Case Matrix" defining all attacks and success metrics, lead the configuration of the complex open-source WIPS, and write the final data-driven comparative analysis report.

Undergraduate Team Roles

  • Red Team Attacker: Systematically executes all defined wireless attacks.
  • Commercial WIPS Analyst: Monitors and documents the performance of the commercial solution.
  • Open Source WIDS Analyst: Monitors and documents the performance of the custom-built solution.

Project 3: The Wi-Fi 7 Performance & Security Testbed

Objective

To build a functional testbed that can demonstrate and measure the key advantages and potential new security considerations of Wi-Fi 7 (802.11be), with a focus on Multi-Link Operation (MLO).

Core Technologies

  • Linux `hostapd` with modern Wi-Fi 6/6E adapters
  • Performance Tools (`iperf3`, `ping`)
  • Packet Analysis (Wireshark)
  • Linux Network Bonding/Teaming

Project Manager Responsibilities

The PM will architect the testbed, lead the effort to simulate MLO by bonding multiple network interfaces, design the performance experiments, and author a research paper on the findings and potential new MLO-related security vulnerabilities.

Undergraduate Team Roles

  • AP Configuration Specialist: Configures multi-band `hostapd` instances.
  • Client & Performance Tester: Implements client-side interface bonding and runs all performance tests.
  • Packet Analyst: Captures and analyzes traffic to verify MLO behavior.

Project 4: The "IoT Secure Zone" Network Architecture

Objective

To design and implement a highly segmented network architecture specifically for insecure or untrusted Internet of Things (IoT) devices. The goal is to allow these devices to function while completely isolating them from the main corporate network and each other.

Core Technologies

  • Enterprise-grade APs (Extreme Networks)
  • Network Access Control (NAC) / RADIUS
  • Firewall (Virtual or Physical)
  • Private Pre-Shared Keys (PPSK) or Client Isolation

Project Manager Responsibilities

The PM will design the overall network architecture, including VLANs and firewall policies, research and select the best client isolation technology, and write a best-practice guide for securing IoT on enterprise Wi-Fi.

Undergraduate Team Roles

  • Wireless Configuration Specialist: Implements the chosen isolation technology on the APs.
  • Firewall & Policy Admin: Configures strict firewall rules for the IoT VLAN.
  • IoT & Penetration Tester: Onboards various IoT devices and attempts to bypass isolation controls.
